# Рекомендации по стилю кода BSL (1С:Предприятие)

## 1. Запросы — всегда в отдельных типизированных функциях

Все запросы к базе данных должны быть написаны в отдельных функциях-конструкторах, которые возвращают типизированный результат (ТаблицаЗначений или РезультатЗапроса).

НЕПРАВИЛЬНО — запрос встроен в бизнес-логику:
```bsl
Процедура ОбработатьТовары()
    Запрос = Новый Запрос;
    Запрос.Текст = "ВЫБРАТЬ Ссылка ИЗ Справочник.Товары";
    Выборка = Запрос.Выполнить().Выбрать();
    Пока Выборка.Следующий() Цикл
        // ...
    КонецЦикла;
КонецПроцедуры
```

ПРАВИЛЬНО — запрос вынесен в отдельную функцию с типизацией:
```bsl
// Возвращаемое значение:
//  ВыборкаИзРезультатаЗапроса:
//   * Ссылка - СправочникСсылка.Товары
Функция ВыбратьТовары()
    Запрос = Новый Запрос;
    Запрос.Текст = "ВЫБРАТЬ Ссылка ИЗ Справочник.Товары";
    Возврат Запрос.Выполнить().Выбрать();
КонецФункции

Процедура ОбработатьТовары()
    Выборка = ВыбратьТовары();
    Пока Выборка.Следующий() Цикл
        // ...
    КонецЦикла;
КонецПроцедуры
```

## 2. Типизация структур — ссылка вместо дублирования

При использовании строгой типизации не дублируйте описание структуры повсюду. Если есть функция-конструктор с полным описанием типов, ссылайтесь на неё с помощью директивы `См.`.

НЕПРАВИЛЬНО — дублирование описания:
```bsl
// Параметры:
//  Данные - Структура:
//   * Товар - СправочникСсылка.Товары
//   * Количество - Число
//   * Цена - Число
Процедура ОбработатьДанные(Данные)
```

ПРАВИЛЬНО — ссылка на конструктор:
```bsl
// Параметры:
//  Данные - См. НовыеДанныеЗаказа
Процедура ОбработатьДанные(Данные)
```

## 3. Функции-конструкторы структур

Избегайте определения структур непосредственно в коде. Используйте функции-конструкторы.

Исключение: простые структуры-фильтры для табличных методов, таких как `НайтиСтроки`, допустимо определять в месте использования.

```bsl
// Возвращаемое значение:
//  Структура:
//   * Товар - СправочникСсылка.Товары
//   * Количество - Число
//   * Цена - Число
//   * Сумма - Число
Функция НовыеДанныеЗаказа()
    Данные = Новый Структура;
    Данные.Вставить("Товар", Справочники.Товары.ПустаяСсылка());
    Данные.Вставить("Количество", 0);
    Данные.Вставить("Цена", 0);
    Данные.Вставить("Сумма", 0);
    Возврат Данные;
КонецФункции
```

## 4. Параметры функций — не более 3-5

Функции должны иметь не более 3-5 параметров. Если требуется больше параметров, группируйте их в типизированную структуру.

НЕПРАВИЛЬНО:
```bsl
Процедура СоздатьДокумент(Товар, Количество, Цена, Склад, Контрагент, Договор, Валюта)
```

ПРАВИЛЬНО:
```bsl
// Параметры:
//  ПараметрыДокумента - См. НовыеПараметрыДокумента
Процедура СоздатьДокумент(ПараметрыДокумента)
```

## 5. Организация кода с помощью областей

Организуйте код в логические области для удобства чтения и навигации.

```bsl
#Область ПрограммныйИнтерфейс

    #Область ДоступКДанным
    // Экспортные функции для работы с данными
    #КонецОбласти

    #Область Вычисления
    // Экспортные функции для вычислений
    #КонецОбласти

#КонецОбласти

#Область СлужебныйПрограммныйИнтерфейс
// Функции, используемые другими модулями внутри подсистемы
#КонецОбласти

#Область СлужебныеПроцедурыИФункции

    #Область Конструкторы
    // Функции-конструкторы (НовыйXxx)
    #КонецОбласти

    #Область Запросы
    // Функции запросов к БД
    #КонецОбласти

#КонецОбласти
```

## 6. Размер функций

Максимальный размер функции — 100-200 строк. Исключения возможны, но их следует избегать.

Большие функции необходимо декомпозировать на более мелкие функции с единой ответственностью.

## 7. Рекомендации по рефакторингу

Не бойтесь рефакторить код при необходимости. Однако всегда консультируйтесь с экспертами перед масштабным рефакторингом.

Рефакторинг необходим, когда:
- Функции превышают 200 строк
- Паттерны кода повторяются 3 и более раз
- Функции имеют более 5 параметров
- Информация о типах неполная

## 8. Именование

- Функции запросов: `ЗапросXxx()` или `ПолучитьДанныеXxx()`
- Конструкторы: `НовыйXxx()` / `НоваяXxx()` / `НовоеXxx()`
- Булевы переменные и функции: начинаются с `Это`, `Есть`, `Показать`, `Включить`
- Документируйте экспортные функции с полной типизацией
- Комментарии объясняют «зачем», а не «что»

## 9. Обработка ошибок

- Проверяйте входные параметры и давайте понятные сообщения об ошибках
- Не подавляйте исключения молча

## 10. Производительность

- Избегайте запросов в циклах
- Используйте пакетные операции
- Группируйте обращения к базе данных
