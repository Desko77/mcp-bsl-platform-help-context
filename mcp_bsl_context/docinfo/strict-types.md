# Строгая типизация BSL

Методические рекомендации по реализации строгой типизации в разработке на 1С:Предприятие 8
с использованием редактора 1C:EDT.

## TOPIC: overview

### Назначение строгой типизации

1. Снижение количества ошибок, обнаруживаемых при разработке
2. Исключение практики написания кода в режиме отладки за счёт максимальной типизации кода для статического анализатора в 1C:EDT
3. Упрощение кода для чтения человеком
4. Типизация кода позволяет активнее использовать различные инструменты анализа кода
5. Улучшение рефакторинга при модификации кода
6. Поддержание документирующих комментариев в актуальном (достоверном) состоянии
7. Применение средств автоматической обработки кода (автоформатирование, автоперевод и т.д.)
8. Упрощение сопровождения кода новыми разработчиками

### Как это работает

При изучении возможностей типизации следует учитывать:

1. Язык исполнения 1С:Предприятие 8 является/остаётся динамически типизированным
2. Все возможности по вычислению типов — это базовые возможности ядра 1C:EDT
3. Плагин «1С:Стандарты разработки V8» лишь добавляет контроль типов в 1C:EDT и проблемных ситуаций на основе базовых возможностей 1C:EDT

### Что такое «строгая типизация»

«Строгая типизация — это контроль (валидация) наличия всех типов, как декларативных, так и динамически вычисляемых, в коде 1С:Предприятие 8.»

Ключевые возможности:
- Контроль наличия типов в точке создания объекта/переменной
- Контроль наличия типов в точке использования (обращение к свойствам или методам объекта)
- Контроль типов возвращаемых значений функций, свойств объектов, переменных
- Запрет изменения типов переменных и свойств объектов
- Пересечение типов при передаче объектов в параметры вызываемых методов
- Контроль декларированных типов и типов из системы Data-flow анализа (DFA) для «пользовательских» объектов данных

#### Чем строгая типизация НЕ является

- Не является ООП
- Не добавляет иерархию типов 1С и не изменяет существующую иерархию
- Не добавляет понятия абстрактных объектов или «Интерфейсов»
- Не изменяет систему типов 1С:Предприятие 8 — только контролирует существующие типы

## TOPIC: enabling

### Включение строгой типизации

Для включения строгой типизации модуля добавьте аннотацию перед первым семантическим объектом (область, процедура, переменная):

```bsl
//@strict-types

#Область ПрограммныйИнтерфейс
...
```

После этого 1C:EDT будет отображать ошибки, если что-то некорректно с объектами и их типами.

**Массовое включение** строгой типизации можно выполнить командой `Включить строгую типизацию (@strict-types) в модулях` из контекстного меню:
- В навигаторе 1С: для проекта в целом, для выбранного списка объектов метаданных
- В обозревателе проектов: для выбранного списка каталогов или файлов модулей
- В редакторе модуля — подменю Исходный текст: для текущего открытого модуля

### Когда применять строгую типизацию

Рекомендуемые случаи:
- Для всех новых конфигураций без наследия нетипизированного кода — включить строгую типизацию для всех модулей
- Для существующих модулей — сначала адаптировать код, затем включить строгую типизацию

## TOPIC: calculated-typing

### Вычисляемая типизация

Динамическое вычисление типов, выполняемое в 1C:EDT на основе контекста 1С:Предприятия и метаданных конфигурации.

#### Отслеживание состояния типа переменной

Система типизации 1C:EDT отслеживает тип переменной в зависимости от места её использования (расположения в коде):

```bsl
МояПеременная = 10;
...
Если МояПеременная > 0 Тогда // Здесь тип — Число
КонецЕсли;

...
МояПеременная = Истина;
...
Если МояПеременная Тогда // Здесь тип — Булево
КонецЕсли;
```

#### Отслеживание состояния типа свойства

При анализе объектов данных (Data-flow анализ), создаваемых программно, система типизации 1C:EDT учитывает только типы, указанные в момент инициализации свойства:

```bsl
Параметры = Новый Структура("МоёСвойство"); // Инициализация без указания начального типа
...
Параметры.МоёСвойство = 10; // Изменение типа с Неопределено на Число НЕ происходит
...
Если Параметры.МоёСвойство > 0 Тогда // Здесь тип МоёСвойство — Неопределено
КонецЕсли;
...
Параметры.Вставить("ДругоеСвойство", Ложь); // Свойство инициализировано с начальным значением

Параметры.ДругоеСвойство = 1; // Изменение типа НЕ происходит
...
Если Параметры.ДругоеСвойство Тогда // Здесь тип свойства — Булево
КонецЕсли;
```

Когда тип значения фактически меняется для свойства объекта, допускающего такое поведение в среде исполнения 1С:Предприятие 8, система типизации и анализа объектов данных 1C:EDT не учитывает это изменение.

Правильный подход: указывать начальное типизированное значение и не допускать изменения типа далее в коде. Два варианта для переменных:
1. Тип явно определён пользователем и далее не меняется; попытки изменения можно отследить
2. Если тип не определён явно, он ВСЕГДА будет неопределённым далее, и плагин будет считать его неопределённым везде

## TOPIC: declarative-typing

### Декларативная типизация

Статическое указание типов в документирующих/типизирующих комментариях, ссылках на функции-конструкторы и ссылках на входящие параметры других методов.

### Структура документирующего комментария

Структура данных документирующего комментария в упрощённом иерархическом виде:

```
Описание:                       # Объект описания, состоящий из коллекции строк и ссылок
  - ТекстоваяЧасть              # Текстовая строка описания
  - ЧастьСсылка                 # Ссылка на веб-страницу или метод конфигурации

СекцияПараметров:                # Секция параметров метода
  ОпределенияПараметров:         # Список параметров
    - ОпределениеПоля:
      Имя: ИмяПараметра          # Имя параметра
      СекцииТипов:               # Список секций типов
        - СекцияТипа:
          ОпределенияТипов:      # Список определений типов
            - ОпределениеТипа:
              ИмяТипа: ИмяТипа   # Имя типа 1С
              ВложенныеТипы:     # Описания типов элементов коллекции
              РасширениеПолей:   # Возможность расширить тип полями

СекцияВозврата:                  # Секция описания возвращаемого значения
  ТипыВозврата:                  # Список секций типов возвращаемого значения
    - СекцияТипа:
      ОпределенияТипов:
        - ОпределениеТипа
```

## TOPIC: type-links

### Ссылки на типы

Система типизации 1C:EDT поддерживает ссылки на типы вместо прямого указания. Используйте ссылки на типы для сокращения объёма текста в документирующих комментариях и повышения точности описания типов.

#### Ссылка на объект табличной части

```bsl
// Параметры:
//  Объект - См. Справочник.Товары.Единицы
Процедура ОбработатьОбъект(Объект)
```

#### Ссылка на строку табличной части

> Такая ссылка пока не поддерживается в 1C:EDT!

```bsl
// Параметры:
//  Объект - СтрокаТабличнойЧасти: См. Справочник.Товары.Единицы
Процедура ОбработатьОбъект(Объект)
```

#### Ссылка на тип реквизита объекта

```bsl
// Параметры:
//  Реквизит1 - См. Справочник.Товары.Артикул
Процедура ОбработатьОбъект(Реквизит1)
```

#### Ссылка на тип реквизита табличной части

```bsl
// Параметры:
//  РеквизитТЧ - См. Справочник.Товары.Единицы.Единица
Процедура ОбработатьОбъект(РеквизитТЧ)
```

#### Ссылка на форму

```bsl
// Параметры:
//  Форма - См. Справочник.Товары.Форма.ФормаЭлемента
Процедура ОбработатьОбъект(Форма)
```

#### Ссылка на элемент или реквизит формы

```bsl
// Параметры:
//  Объект - См. Справочник.Товары.Форма.ФормаЭлемента.Объект
Процедура ОбработатьОбъект(Объект)
```

```bsl
// Параметры:
//  Список - См. Справочник.Товары.Форма.ФормаСписка.Элементы.Список
Процедура ОбработатьОбъект(Список)
```

#### Ссылка на тип параметра метода

Из модуля менеджера:
```bsl
// Параметры:
//  Список - См. Справочники.Товары.МетодМодуляМенеджера.Параметр1
Процедура ОбработатьОбъект(Список)
```

Из модуля объекта:
```bsl
// Параметры:
//  Список - См. СправочникОбъект.Товары.МетодМодуляОбъекта.Параметр2
Процедура ОбработатьОбъект(Список)
```

#### Ссылка на функцию-конструктор

```bsl
// Параметры:
//  Объект - См. НовыйОбъектДанных
Процедура ОбработатьОбъект(Объект)
```

## TOPIC: local-variables

### Инициализация локальных переменных

- ЗАПРЕЩЕНО: Инициализировать переменные через `Перем`, так как такая переменная инициализируется с типом `Неопределено`, и дальнейшее изменение типа может быть не видно статическому анализатору

НЕПРАВИЛЬНО:
```bsl
Процедура ОбработатьТекущуюСтроку(ТЧ)
    Перем ТекущаяСсылка;

    Для Каждого СтрокаТЧ Из ТЧ Цикл
        ТекущаяСсылка = СтрокаТЧ.Ссылка;
        ...
    КонецЦикла;
    Объект = ТекущаяСсылка.ПолучитьОбъект();
...
```

ПРАВИЛЬНО:
```bsl
Процедура ОбработатьТекущуюСтроку(ТЧ)
    ТекущаяСсылка = Справочники.Товары.ПустаяСсылка();

    Для Каждого СтрокаТЧ Из ТЧ Цикл
        ТекущаяСсылка = СтрокаТЧ.Ссылка;
        ...
    КонецЦикла;
    Объект = ТекущаяСсылка.ПолучитьОбъект();
...
```

- ЗАПРЕЩЕНО: Инициализировать переменные внутри циклов или условий с последующим использованием вне циклов/условий
- Уточнение типа для локальной переменной, инициализированной функцией, возвращающей более общий тип, возможно через встроенное указание типа:

ПРАВИЛЬНО:
```bsl
МойОбъект = ОбщийМодуль.СоздатьНовыйОбъектПоТипу(ТипОбъекта); // СправочникОбъект.Товары -
```

Для сложных объектов данных используйте ссылки на функции-конструкторы:
```bsl
МойОбъект = Параметры.МойОбъект; // См. НовыйОбъектДанных
```

## TOPIC: module-variables

### Инициализация переменных модуля

Переменные модуля объекта, формы, конфигурации, включая глобальные (экспортные) переменные, должны быть объявлены со статическим указанием типа в комментарии. Также инициализируйте переменную модуля начальным/пустым значением в коде модуля.

НЕПРАВИЛЬНО:
```bsl
#Область ОписаниеПеременных

&НаКлиенте
Перем КэшированныеЗначения; //используется механизмом обработки изменений ТЧ

&НаКлиенте
Перем ИдентификаторТекущихДанных; //используется для передачи текущей строки в обработчик ожидания

&НаКлиенте
Перем ПараметрыДляЗаписи Экспорт;

#КонецОбласти
```

ПРАВИЛЬНО:
```bsl
#Область ОписаниеПеременных

//используется механизмом обработки изменений ТЧ
&НаКлиенте
Перем КэшированныеЗначения; // См. РаботаСТабличнымиЧастямиКлиентСервер.ПолучитьСтруктуруКэшированныхЗначений

//используется для передачи текущей строки в обработчик ожидания
&НаКлиенте
Перем ИдентификаторТекущихДанных; // Число -

&НаКлиенте
Перем ПараметрыДляЗаписи Экспорт; // Структура -

#КонецОбласти

....

ИдентификаторТекущихДанных = -1;
ПараметрыДляЗаписи = Новый Структура;
```

## TOPIC: structure-keys

### Инициализация ключей структуры

- Значения ключей структуры (как в конструкторе, так и при добавлении в структуру) должны быть немедленно инициализированы пустым значением типа, который будет использоваться далее
- Изменение типа значения ключа структуры НЕ допускается

НЕПРАВИЛЬНО:
```bsl
Процедура Обработка()
    ОбъектСправочника = Справочники.Товары.СоздатьЭлемент();
...
    Параметры = Новый Структура("Ссылка");
    Параметры.Ссылка = ОбъектСправочника.Ссылка;
```

ПРАВИЛЬНО:
```bsl
Процедура Обработка()
    ОбъектСправочника = Справочники.Товары.СоздатьЭлемент();
...
    Параметры = Новый Структура("Ссылка", Справочники.Товары.ПустаяСсылка());
    Параметры.Ссылка = ОбъектСправочника.Ссылка;
```

ПРАВИЛЬНО:
```bsl
Процедура Обработка()
    ОбъектСправочника = Справочники.Товары.СоздатьЭлемент();
...
    Параметры = Новый Структура;
    Параметры.Вставить("Ссылка", ОбъектСправочника.Ссылка);
```

- При добавлении ключа в существующую структуру пишите код «прозрачно» для статического анализатора, чтобы вставка ключа и его использование были в одной области видимости

## TOPIC: arrays

### Описание массивов

- Как правило ЗАПРЕЩЕНО: Создавать массив `Новый Массив;` без указания типа его значений, если далее в текущем модуле есть обращение к элементам массива

НЕПРАВИЛЬНО:
```bsl
    СписокСсылок = Новый Массив;
    СписокСсылок.Добавить(Ссылка);
...
    Для Каждого Ссылка Из СписокСсылок Цикл
        Объект = Ссылка.ПолучитьОбъект();
...
```

ПРАВИЛЬНО:
```bsl
    СписокСсылок = Новый Массив; // Массив из СправочникСсылка.Товары -
    СписокСсылок.Добавить(Ссылка);
...
    Для Каждого Ссылка Из СписокСсылок Цикл
        Объект = Ссылка.ПолучитьОбъект();
...
```

ПРАВИЛЬНО (с использованием функции-конструктора):
```bsl
    СписокСсылок = НовыйСписокТоваров();
    СписокСсылок.Добавить(Ссылка);
...
    Для Каждого Ссылка Из СписокСсылок Цикл
        Объект = Ссылка.ПолучитьОбъект();
...

// Возвращаемое значение:
//  Массив из СправочникСсылка.Товары
Функция НовыйСписокТоваров()
    Возврат Новый Массив;
КонецФункции
```

- Не рекомендуется: использовать объекты разных типов как значения (строки с числами, простые типы со ссылочными, объекты БД со структурами и т.д.)
- Если значение массива сложное, используйте функцию-конструктор для инициализации пустого массива или массива с данными
- В документирующих комментариях указывайте тип элемента массива для параметров и возвращаемых значений

## TOPIC: value-table

### Описание ТаблицыЗначений и ДереваЗначений

- При описании объектов `ТаблицаЗначений`, `ДеревоЗначений` всегда описывайте все колонки и их типы

ПРАВИЛЬНО:
```bsl
// Возвращаемое значение:
// ТаблицаЗначений:
//  * Товар - СправочникСсылка.Товары
//  * Характеристика - СправочникСсылка.ХарактеристикиТоваров
//  * Склад - СправочникСсылка.Склады
//  * Количество - Число
// ...
Функция ТоварыДляСписания()
....
    Возврат Таблица;
КонецФункции
```

- Как правило ЗАПРЕЩЕНО: использовать описание типа `ТаблицаЗначений` как входящий параметр экспортного метода с описанием колонок. Используйте ссылку на функцию-конструктор, создающую эту таблицу

НЕПРАВИЛЬНО:
```bsl
// Параметры:
//  Таблица - ТаблицаЗначений
//   * Товар - СправочникСсылка.Товары
Процедура ОбработатьТаблицу(Таблица) Экспорт
    Для Каждого СтрокаТаблицы Из Таблица Цикл
        СтрокаТаблицы.Товар = Справочники.Товары.ПустаяСсылка();
....
```

ПРАВИЛЬНО:
```bsl
// Параметры:
// Таблица - См. ТоварыДляСписания
Процедура ОбработатьТаблицу(Таблица)
    Для Каждого СтрокаТаблицы Из Таблица Цикл
        СтрокаТаблицы.Товар = Справочники.Товары.ПустаяСсылка();
....
```

## TOPIC: table-row

### Описание строки ТаблицыЗначений или ДереваЗначений

Используйте один из двух вариантов:

**Вариант 1: Прямое указание колонок:**

НЕПРАВИЛЬНО:
```bsl
// Параметры:
// СтрокаТаблицы - СтрокаТаблицыЗначений
```

## TOPIC: map

### Описание Соответствия

- Соответствие — это сложный тип с фиксированным набором колонок (ключ и значение). Типы по умолчанию для ключа и значения — `Произвольный`; создайте функцию-конструктор, которая инициализирует соответствие и описывает типы ключа и значения
- Избегайте комбинирования разных типов для ключа и значения
- Нельзя указать типы ключа и значения для инициализированной переменной во встроенном комментарии
- При получении объекта данных из контейнера используйте ссылку на функцию-конструктор

НЕПРАВИЛЬНО:
```bsl
   МояПеременная = Новый Соответствие; // Соответствие из Строка - так не работает
```

ПРАВИЛЬНО:
```bsl
// Возвращаемое значение:
// Соответствие из КлючИЗначение:
// * Ключ - Строка
// * Значение - Массив из ОбъектДокумента
Функция НовоеСоответствие()
   ...
```

ПРАВИЛЬНО:
```bsl
// Создаём данные с помощью функции-конструктора и заполняем
Данные = НовоеСоответствие();
...
ПоместитьВоВременноеХранилище(Адрес, Данные);
...

// При инициализации переменной указываем ссылку на оригинальную функцию, создавшую типизированный объект
МояПеременная = ПолучитьЗначениеИзХранилища(Адрес); // См. НовоеСоответствие
   ...
```

## TOPIC: type-narrowing

### Сужение типа локальной переменной или параметра

Вы можете безопасно сузить (или фактически установить для статического анализатора) тип локальной переменной метода, входящего параметра или переменной модуля через проверку типа:

```bsl
Если ТипЗнч(МояПеременная) = Тип("СправочникОбъект.Товары") Тогда
    МояПеременная.Артикул = "";
    МояПеременная.Записать();
```

Внутри условия переменная будет иметь тип, указанный в проверке, как в среде исполнения, так и для статического анализатора. Условие проверки должно быть простым.

Для задачи переопределения вычисленных типов на основе кода, который вычисляет система 1C:EDT, укажите тип входящего параметра в документирующих комментариях:

```bsl
// @strict-types

...
// Где-то в коде — вызов функции ниже
Ответ = ШаблонПечати(Документы.Заказ.СоздатьДокумент());
...

// Параметры:
//  Док - ДокументОбъект.Заказ - Это строка
//
// Возвращаемое значение:
//  ТабличныйДокумент - Тест
&НаСервереБезКонтекста
Функция ШаблонПечати(Док)
    Док.Автор = Справочники.Пользователи.ПустаяСсылка();
    Макет = Док.ВернутьМакет();
    Возврат Макет;
КонецФункции
```

- Допустимо: описывать часть формы, общую для нескольких форм или реализацию какого-либо интерфейса для общего механизма конфигурации

ПРАВИЛЬНО:
```bsl

// Форма - ФормаКлиентскогоПриложения:

    ТекущиеДанные = Форма.Элементы.Товары.ТекущиеДанные;
....
```

- Если используется конкретная форма, укажите полную ссылку на форму в документирующих комментариях:

```bsl
// Параметры:
// Форма - См. Справочник.Товары.Форма.ФормаЭлемента
Процедура ПриСозданииНаСервере(Форма)
```

#### Получение и открытие формы

- При использовании методов `ПолучитьФорму()` и `ОткрытьФорму()` с присвоением возвращаемого значения формы после открытия — укажите строковый литерал с полным именем формы в качестве первого параметра
- НЕ выносите строковый литерал в отдельную переменную, так как в этом случае переменная формы будет содержать общий тип `ФормаКлиентскогоПриложения`, а не конкретный тип формы справочника товаров

НЕПРАВИЛЬНО:
```bsl
ИмяФормы = "Справочник.Товары.Форма.ФормаЭлемента";
Форма = ПолучитьФорму(ИмяФормы);
```

ПРАВИЛЬНО:
```bsl
Форма = ПолучитьФорму("Справочник.Товары.Форма.ФормаЭлемента");
// или
Форма = Справочники.Товары.ПолучитьФорму("ФормаЭлемента");
```

## TOPIC: string-literals

### Использование строковых литералов в качестве имён

НЕ обращайтесь к элементам именованной коллекции (например, `ВсеЭлементыФормы`) через строковый индекс с именем элемента. Вместо этого обращайтесь к элементу напрямую, так как статический анализатор может контролировать наличие элемента и его тип.

НЕПРАВИЛЬНО:
```bsl
Элементы["Наименование"].Видимость = Ложь;
```

ПРАВИЛЬНО:
```bsl
Элементы.Наименование.Видимость = Ложь;
```

Исключения:
* Обращение к программно созданным элементам, не существующим в момент статического анализа. Укажите тип локальной переменной и затем обращайтесь к её свойствам:

```bsl
Элемент = Элементы["Наименование"]; // ПолеФормы -
Элемент.Видимость = Ложь;
Элемент.Доступность = Истина;
```

* Наличие элемента в коллекции может быть необязательным, с проверкой наличия элемента:

```bsl
    Элемент = Элементы.Найти("Наименование");
    Если Элемент <> Неопределено Тогда
        Элемент.Видимость = Ложь;
...
```

## TOPIC: query-results

### Выборка и выгрузка результатов запроса

При типизации выборки/выгрузки результатов запроса используйте возможности:

1. Статическое описание всех полей выборки в возвращаемом значении функции-получателя данных. Применимо для функций, динамически формирующих текст запроса и выборку, а также для всех экспортных функций, возвращающих выборку.
2. Динамическая типизация полей на основе текста запроса. В будущих версиях 1C:EDT может быть реализована возможность обращения к полям запроса, когда текст запроса и выборка находятся в одной процедуре, текст запроса не содержит ошибок, выборка не передаётся в другой модуль.

НЕПРАВИЛЬНО:
```bsl
Запрос = Новый Запрос;
Запрос.Текст =
    ТекстЗапросаОстатков()
    + ТекстЗапросаРезервов();
РезультатЗапроса = Запрос.Выполнить();
Выборка = РезультатЗапроса.Выбрать();
Пока Выборка.Следующий() Цикл
    Ссылка = Выборка.Ссылка.ПолучитьОбъект();
...
```

ПРАВИЛЬНО:
```bsl
// Возвращаемое значение:
// ВыборкаИзРезультатаЗапроса:
//  * Товар - СправочникСсылка.Товары
Функция ОстаткиДляОбработки()
    Запрос = Новый Запрос;
    Запрос.Текст =
        ТекстЗапросаОстатков()
        + ТекстЗапросаРезервов();
    РезультатЗапроса = Запрос.Выполнить();
    Возврат РезультатЗапроса.Выбрать();
КонецФункции;

...

Выборка = ОстаткиДляОбработки();
Пока Выборка.Следующий() Цикл
    Ссылка = Выборка.Товар.ПолучитьОбъект();
...
```

## TOPIC: documentation-syntax

### Синтаксис документирующих комментариев

Описание — многострочное, со ссылками:
```bsl
// В описании рекомендуется все ссылки писать на отдельной строке.
// ссылки на веб-страницы:
// См. https://1c.ru
// Ссылка на функцию в тексте описания
// См. ОбщийМодуль.ФункцияМодуля
//
Процедура ОбработкаОбъекта(Объект)
```

В секции параметров после ключевого слова `Параметры` — двоеточие ОБЯЗАТЕЛЬНО:
```bsl
// Описание метода
//
// Параметры:
//  Объект - СправочникОбъект.Товары
Процедура ОбработкаОбъекта(Объект)
```

Несколько типов на отдельных строках с дефисом:
```bsl
// Параметры:
//  Объект - СправочникОбъект.Товары - Описание для типа
//         - ДокументОбъект.Продажи - Описание для другого типа
Процедура ОбработкаОбъекта(Объект)
```

Несколько типов на одной строке — через запятую:
```bsl
// Параметры:
//  Объект - СправочникОбъект.Товары, ДокументОбъект.Продажи - Описание для списка типов
Процедура ОбработкаОбъекта(Объект)
```

Расширение полей для типа, **строго после двоеточия** в конце однострочного описания:
```bsl
// Параметры:
//  Объект - Структура - Здесь не может быть многострочного описания:
//  * ПолеСтруктуры - ТаблицаЗначений - Описание поля структуры, расширение для таблицы:
//    ** ИмяКолонки - Число - описание колонки таблицы
Процедура ОбработкаОбъекта(Объект)
```

Можно опустить описание, оставив только типы и обязательное двоеточие:
```bsl
// Параметры:
//  Объект - Структура:
//  * ПолеСтруктуры - ТаблицаЗначений:
//    ** ИмяКолонки - Число
Процедура ОбработкаОбъекта(Объект)
```

Тип элемента коллекции в параметрах:
```bsl
// Параметры:
//  Объект - Массив из СправочникОбъект.Товары - Здесь тип одного элемента
Процедура ОбработкаОбъекта(Объект)
```

Составной тип элемента — через запятую:
```bsl
// Параметры:
//  Объект - Массив из СправочникОбъект.Товары, ДокументОбъект.Продажи -
Процедура ОбработкаОбъекта(Объект)
```

Список типов возвращаемого значения — каждый на новой строке:
```bsl
// Возвращаемое значение:
//  - Структура
//  - ТаблицаЗначений
Функция ОбработкаОбъекта(Объект)
```

Ссылка на функцию-конструктор в возвращаемом значении:
```bsl
// Возвращаемое значение:
//  См. НовыйОбъектДанных
Функция ОбработкаОбъекта(Объект)
```

## TOPIC: diagnostics

### Как диагностировать нетипизированный код

Диагностика проблемы:
- При наведении мыши на объект в коде — всплывающая подсказка не показывает тип, значит система типизации 1C:EDT не смогла его вычислить
- В коде после точки при обращении к свойству/вызову метода подсказка ввода (нажмите `Ctrl+Space`) — подсказка не показывает свойства в формате `Объект.Свойство <ТипСвойства> ~ ТипОбъекта`

Для поиска причин нетипизированного кода:
1. Найдите самый первый объект в цепочке вызовов в данной строке
2. Определите, есть ли у этого объекта тип — наведите мышь или нажмите `F2` для подсказки
3. Перейдите к определению объекта — нажмите `F3` и в месте определения определите тип
4. Тип определён слишком обще (абстрактно), например `УправляемаяФорма`, `ТаблицаЗначений`, `Массив`, `Структура`, `ДокументОбъект`, `СправочникСсылка` и т.д. без указания конкретных свойств для обращения
5. Правильно ли написан типизирующий документирующий комментарий
6. Если типы корректны — от места определения объекта/переменной до места потери типа — проверьте все места использования: переприсвоение, передачу в вызов метода и т.д.
